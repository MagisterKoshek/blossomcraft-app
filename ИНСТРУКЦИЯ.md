# BlossomCraft — приложение на Java (инструкция)

Это **родное (native) приложение BlossomCraft на Java**, повторяющее сайт
«один в один»: вход (логин / регистрация / Google / сессия по токену), магазин,
музыка с плеером, короткие видео, мессенджер в стиле Telegram (личные чаты +
группы + каналы) с **управлением группами/каналами и админ-панелью**, профиль и
все четыре темы сайта (тёмная / светлая / зеркальная / серо-зеркальная).

Приложение работает **точно как сайт**: оно обращается к тому же PHP-API
(`/api/*.php`), который уже обслуживает сайт. Никакой отдельной логики на сервере
не нужно — приложение это просто второй «клиент» вашего сайта.

Платформы:
- **ПК (Desktop)** — Windows, macOS, Linux (JavaFX).
- **Телефон** — Android.
- **iOS** — возможно через Gluon (сборка только на Mac). См. `iOS-GLUON.md`.

> Replit не умеет показывать родные приложения, поэтому проект отдаётся как
> **исходный код, готовый к сборке**. Соберите его по шагам ниже на своём
> компьютере (Windows/Mac/Linux) или в Android Studio.

## 0. Самый простой способ получить готовые .apk и .exe (автосборка)

Готовые файлы под **Windows (.exe)** и **Android (.apk)** невозможно собрать на
сервере Replit (там Linux, нет Windows и Android-инструментов, и нет
кросс-компиляции). Поэтому в проект добавлена **бесплатная облачная автосборка
GitHub Actions** — она соберёт всё за вас, и вам не нужны ни Java, ни Android
Studio на своём компьютере.

1. Заведите бесплатный аккаунт на <https://github.com> и нажмите
   **New repository** (имя любое, можно Private), создайте репозиторий.
2. Загрузите в него **содержимое папки `BlossomCraftApp`** так, чтобы файл
   `settings.gradle` и папка `.github` оказались в **корне** репозитория.
   Проще всего: на странице репозитория — **Add file → Upload files** — и
   перетащите туда все файлы и папки из распакованного архива (включая скрытую
   папку `.github`).
3. Откройте вкладку **Actions** (Действия) → слева выберите
   **Build BlossomCraft apps** → кнопка **Run workflow**. Сборка идёт ~5–10 минут.
4. Когда напротив запуска появится зелёная галочка — откройте его и внизу, в
   разделе **Artifacts**, скачайте:
   - **BlossomCraft-Android-APK** — файл `.apk`. Скопируйте на телефон и
     установите (разрешив установку из неизвестных источников).
   - **BlossomCraft-Windows** — архив с программой. Распакуйте и запустите
     **`BlossomCraft.exe`** (внутри может также лежать установщик `.msi`).

Адрес вашего сайта (`https://bc-shop.duckdns.org/api`) **уже вшит** в сборку —
приложение сразу работает с той же базой данных, что и сайт (аккаунты, сообщения,
магазин — общие). Изменить адрес при желании можно прямо в приложении
(«Настройки» на ПК / «Профиль» на Android).

> iOS в автосборку не входит по вашему запросу. Для iOS нужен Mac с Xcode и
> платный аккаунт Apple Developer — см. `iOS-GLUON.md`, если понадобится позже.

## 1. Куда вписать адрес сайта (URL)

Приложению нужно знать **адрес вашего API** — это папка на хостинге, где лежат
`auth.php`, `shop.php`, `music.php` и т. д. (папка `api/` сайта). Например:
`https://ваш-сайт.ru/api`.

Адрес уже **вписан на ваш сайт** (`https://bc-shop.duckdns.org/api`) — приложение
сразу работает с той же базой, что и сайт. Изменить его при необходимости можно
одним из способов:

1. **Один раз для всех сборок** — откройте файл `gradle.properties` и впишите:

   ```properties
   blossomcraft.api.base=https://ваш-сайт.ru/api
   ```

2. **Прямо в приложении** (можно поменять в любой момент):
   - **ПК:** экран **«Настройки»** → поле адреса API → «Сохранить».
   - **Android:** вкладка **«Профиль»** → поле адреса API → «Сохранить».

3. **При запуске одной командой** (не меняя файлы) — см. команды ниже
   (`-Dblossomcraft.api.base=...` для ПК, `-Pblossomcraft.api.base=...` для
   Android).

## 2. Где данные для базы данных (ВАЖНО)

**Данные доступа к базе данных в приложение НЕ вписываются и в нём не хранятся.**

Приложение — это клиент: оно общается с сайтом по интернету (HTTP) и **никогда не
подключается к MySQL напрямую**. Логин/пароль/имя базы остаются **только на
хостинге** в конфиге сайта (обычно `api/config.php`), как и сейчас у сайта.

Итого:
- **В приложении** задаётся только **адрес API** (см. раздел 1).
- **На хостинге** в `api/config.php` остаются настройки базы (хост, имя БД,
  пользователь, пароль) — вы их не трогаете и никому не передаёте.

Это безопасно: пароль от базы не попадает на телефоны/ПК пользователей.

## 3. Что нужно установить (один раз)

- **JDK 17 или новее** (рекомендуется Temurin/Adoptium). Для создания
  установщиков (раздел 6) удобнее **JDK 21**.
- **Gradle 8.x** — система сборки. Можно установить отдельно, либо один раз
  сгенерировать «обёртку»:

  ```bash
  gradle wrapper --gradle-version 8.7
  ```

  после чего вместо `gradle` использовать `./gradlew` (на Windows — `gradlew.bat`).
- **Только для Android:** **Android Studio** (версия Giraffe и новее). Она сама
  поставит Android SDK (API 34).

## 4. ПК: сборка и запуск (Windows / macOS / Linux)

JavaFX подтягивается автоматически — отдельно ничего ставить не нужно.

```bash
cd BlossomCraftApp

# Просто запустить приложение:
gradle :desktop:run -Dblossomcraft.api.base=https://ваш-сайт.ru/api

# Или собрать готовую папку с программой:
gradle :desktop:installDist
# Результат: desktop/build/install/desktop/bin/  (там запускающие скрипты)
```

На Windows, если используете обёртку, команды те же, но через `gradlew.bat`,
например: `gradlew.bat :desktop:run`.

Токен входа сохраняется (как `localStorage` на сайте), поэтому при следующем
запуске приложение само восстановит сессию. Тему и адрес API можно менять в
**«Настройках»**.

## 5. Android: сборка и запуск

**Через Android Studio (проще всего):**

1. Откройте папку `BlossomCraftApp/` в Android Studio.
2. Дождитесь синхронизации Gradle (она сама создаст `local.properties` с путём к
   SDK).
3. Укажите адрес API: впишите `blossomcraft.api.base` в `gradle.properties`
   (раздел 1) — или поменяете прямо в приложении на вкладке «Профиль».
4. Нажмите ▶ и запустите конфигурацию `android` на эмуляторе или телефоне.

**Через командную строку:**

```bash
cd BlossomCraftApp
# Должен быть установлен Android SDK (API 34) и задан ANDROID_HOME
gradle :android:assembleDebug -Pblossomcraft.api.base=https://ваш-сайт.ru/api
# Готовый файл: android/build/outputs/apk/debug/android-debug.apk
gradle :android:installDebug   # установить на подключённый телефон/эмулятор
```

Файл `android-debug.apk` можно скопировать на телефон и установить (разрешив
установку из неизвестных источников).

> Вход по e-mail/паролю и по токену работает сразу. Для кнопки «Google» нужно
> прописать OAuth-ключ — подробности в `README.md` (раздел «Google sign-in»).

## 6. Установщики с «вшитой» Java (по желанию)

`installDist` (раздел 4) делает папку, но требует, чтобы на компьютере была
установлена Java. Чтобы отдать пользователям **самодостаточный установщик**
(Java внутри), используйте `jpackage` (входит в JDK). Точные команды для
Windows (.msi/.exe), macOS (.dmg) и Linux (.deb) — в `README.md`, раздел
**«Native installers»**. Важно: установщик под каждую систему собирается **на
этой же системе** (Windows-установщик — на Windows и т. д.).

## 7. iOS (по возможности)

iOS достижим через **Gluon** (GraalVM + Gluon Substrate), потому что общий модуль
`:core` — это обычная Java, а интерфейс на JavaFX. **Собирать можно только на
Mac с Xcode** — на Replit и на Windows это невозможно. Готовый пример
конфигурации и команды — в файле **`iOS-GLUON.md`**.

Честная оговорка: бизнес-логика (вход, магазин, музыка, видео, сообщения, админ)
переносится на iOS как есть; экраны JavaFX, возможно, потребуют косметической
адаптации под сенсорный экран.

## 8. Что уже сделано «как на сайте»

- Вход: логин / регистрация / Google / сессия по токену; авто-выход при 401.
- Магазин, музыка (с плеером), короткие видео.
- Сообщения: личные чаты, группы и каналы.
- **Управление группами/каналами** (для владельца/админа): изменение
  названия и описания, код приглашения (с обновлением), список участников с
  назначением/снятием админа и удалением, удаление группы/канала (владелец).
  - ПК: кнопка **«⚙ Настройки»** на вкладке групп.
  - Android: меню **«Управление»** (три точки) внутри группы.
- **Админ-панель** (у кого есть право `manage_users`): список пользователей с
  переименованием, сменой роли и удалением.
  - ПК: раздел «Админ».
  - Android: кнопка **«Админ-панель»** на вкладке «Профиль».
- Все четыре темы сайта.

Права проверяются и на сервере — приложение лишь показывает доступные действия.

## 9. Частые вопросы

- **Нужно ли что-то менять на сервере?** Нет. Приложение использует тот же API,
  что и сайт.
- **Где взять адрес API?** Это адрес папки `api/` вашего сайта, например
  `https://ваш-сайт.ru/api`.
- **Безопасно ли это?** Да — пароль от базы остаётся на хостинге в `config.php`
  и в приложение не попадает.
- **Я на Windows и не могу собрать сайт (npm build) — это другое?** Да. Здесь
  сборка идёт через **Gradle/Android Studio**, а не npm. Достаточно установить
  JDK (и Android Studio для телефона).
